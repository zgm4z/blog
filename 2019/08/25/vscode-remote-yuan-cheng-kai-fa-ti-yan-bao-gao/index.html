vscode remote 远程开发体验报告

北京时间 2019 年 5 月 3 日 微软发布了 vscode remote 插件，正式宣布远程开发进入具有与本地开发体验一致的时代，本次发布的插件可以工作在 虚拟机，远程主机，容器和 WSL 环境中，只需要安装相应插件即可，至本文发布之时，已经可以在 stable 的 vscode 上安装 vscode remote 插件，搜索 vscode remote 即可安装。

博主由于毕设需要开发一个和 linux 内核通信的应用，但是手里并没有多余的机器，于是就上了 vscode remote 的船，本次的开发方案是 vscode remote + ssh，在博主的 MBP 上使用虚拟机安装了 Ubuntu 然后开启 SSH 服务并在本地 vscode 安装远程开发插件，配置完成之后就可以愉快的打开远程机器的项目啦~

如上图所示，就是配置好了之后的界面，不仅支持远程连接，还可以做端口转发，再来一张打开远程项目的图：

与本地项目基本一样，并且可以在终端 Tab 里打开远程主机的终端连接，debug 功能也很完美。

当然如果主机在远程的话需要较好的网络环境才能具有比较快的响应速度，像自动完成，代码提示的功能，不过像博主这样的方案基本上感觉不到延迟了，与本地开发并无二致。
